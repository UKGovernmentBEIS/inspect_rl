# 003 Operational Improvements ‚Äî Session 1

## Problems

1. **1 unique prompt per step**: `per_device_train_batch_size=8` with `num_generations=8` means TRL's RepeatSampler yields 8 copies of the same prompt per step. 200 steps = 200 unique prompts out of 5,000. Terrible coverage.

2. **No validation**: Training runs blind ‚Äî no held-out eval to detect overfitting or measure progress on unseen problems.

3. **Event loop crash with multi-turn agents**: `eval_async()` inside `asyncio.run()` broke Inspect's internal async machinery, causing `RuntimeError('no running event loop')` for `basic_agent` samples.

4. **Noisy Inspect display**: The rich TUI output from Inspect cluttered training logs.

## Changes

### Batch size fix (`gsm8k.py`)
- `per_device_train_batch_size`: 8 ‚Üí 16 (user tuned down from my initial 64)
- Now 2 unique prompts per step instead of 1. Still conservative on memory.

### Train/val split (`gsm8k.py`)
- `get_task(split=)` now parameterized ‚Äî train split (7,473) for training, test split (1,319) for validation
- New CLI params: `--eval_steps=25`, `--eval_limit=50`

### Periodic eval callback (`trainer.py`)
- `_InspectEvalCallback(TrainerCallback)` runs Inspect eval every N steps
- Uses temperature=0 for deterministic eval on held-out samples
- Extracts metrics from `log.results.scores`, prints `[eval step N] eval/scorer/metric=X.XXXX`
- Logs to wandb if configured
- `fail_on_error=False` so eval failures don't crash training
- Val eval logs go to `run_dir/val_eval_logs/` (separate from training rollout logs)
- `inspect_rl_train()` extended with `eval_task`, `eval_steps`, `eval_limit` params

### Sync eval fix (`rollout.py`)
- Switched from `eval_async()` + `asyncio.run()` to sync `inspect_eval()` 
- Fixes `RuntimeError('no running event loop')` in multi-turn agent samples
- Also fixed `asyncio.get_event_loop()` ‚Üí `asyncio.get_running_loop()` in `trl_vllm_provider.py`

### Display cleanup (`rollout.py`)
- `display="plain"` instead of rich TUI ‚Äî cleaner output in training logs

### Interrupt handling (`trainer.py`)
- Ctrl-C during training saves `checkpoint-interrupted` before exiting
